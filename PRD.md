# AutoGLM 隐私合规智检 - 产品需求文档 (PRD)

## 1. 项目背景 (Project Background)
随着《个人信息保护法》(PIPL)、《数据安全法》(DSL) 等法律法规的实施，监管部门对移动应用（App）的隐私合规要求日益严格。企业法务、安全及产品团队面临着巨大的合规压力，人工审核隐私政策不仅效率低下，而且容易遗漏细节。
为了解决这一痛点，本项目旨在开发一款基于大模型（AutoGLM）的隐私合规智能检测工具，能够自动化分析隐私政策文本，实时监测行业合规动态，帮助企业快速识别风险并提供整改建议。

## 2. 项目目标 (Project Goals)
1.  **自动化合规检测**：利用大模型能力，对用户上传的隐私政策文档或文本进行深度分析，自动识别违规条款。
2.  **实时情报监测**：聚合全网合规快讯（Google News RSS）及头部应用（微信、抖音等）的隐私政策变更预警。
3.  **行业对标参考**：提供主流 App 的协议库，方便用户快速对标行业头部产品的合规实践。
4.  **提升审核效率**：将传统的人工审核转化为"AI 初审 + 人工复核"模式，大幅降低合规成本。

## 3. 目标人群 (Target Audience)
*   **法务/合规人员**：需要快速审核大量隐私协议，确保产品符合最新法律法规。
*   **安全工程师**：关注数据安全风险，需要监测第三方 SDK 及数据收集行为的合规性。
*   **产品经理 (PM)**：在产品设计阶段需要参考竞品隐私策略，确保功能设计合规。
*   **App 开发者**：需要了解最新的合规要求，避免因违规导致应用下架或处罚。

## 4. 交互逻辑 (Interaction Logic)

### 4.1 核心页面结构
*   **顶部导航栏 (Header)**：显示项目名称 "AutoGLM 隐私合规智检"，提供模型配置入口。
*   **合规快讯栏 (Notice Bar)**：
    *   滚动展示最新的合规新闻（来源：Google News RSS）。
    *   展示重点监测 App 的隐私政策更新预警（模拟监测）。
    *   支持点击查看详情或跳转原文。
*   **主操作区 (Input Area)**：
    *   **文本输入**：支持直接粘贴隐私政策文本。
    *   **文件上传**：支持 PDF、Word、TXT 格式文件上传（限制 10MB/5个文件）。
    *   **协议库导入**：点击"行业协议库"可选择头部 App，自动填充其协议链接和检测提示词。
*   **检测结果页 (Result Card)**：
    *   **风险概览**：显示整体风险等级（高/中/低）。
    *   **详细条目**：列出具体的违规点，包含：
        *   **法规依据**：引用的具体法律条款（如 PIPL 第六条）。
        *   **原文引用**：违规的协议原文片段。
        *   **风险原因**：AI 生成的风险解释。
        *   **整改建议**：针对性的 3 条修改建议。
        *   **行业参考**：对比行业均值或标准（如"行业均值：≥95% 授权率"）。

### 4.2 关键业务流程
1.  **配置模型**：用户首次使用需点击设置，输入火山引擎/AutoGLM 的 API Key 和 Endpoint ID。默认模型名称为 "AutoGLM"。
2.  **发起检测**：
    *   用户输入文本或上传文件 -> 点击"开始合规检测"。
    *   系统进入"加载中"状态，展示动态动画（扫描、解析中）。
3.  **查看结果**：
    *   检测完成后，自动切换至结果视图。
    *   用户可切换查看不同文件/App 的检测结果。
    *   支持点击"返回"修改输入内容。

## 5. 验收标准 (Acceptance Criteria)

### 5.1 功能验收
*   [ ] **模型配置**：能够正确保存和读取 API Key、Endpoint ID，默认模型名为 AutoGLM。
*   [ ] **合规快讯**：
    *   能够成功从 Google News RSS 获取包含"隐私合规"、"数据安全"等关键词的新闻。
    *   能够模拟展示指定 App（如微信、美团）的策略更新预警。
    *   点击新闻链接能正确跳转。
*   [ ] **文件解析**：支持 .pdf, .docx, .txt 文件上传并提取文本内容。
*   [ ] **智能检测**：
    *   调用 LLM 接口成功，返回符合 JSON Schema 定义的数据。
    *   能够准确识别"高风险"项（如敏感个人信息未授权收集）。
*   [ ] **协议库**：点击协议库中的 App 图标，能自动将该 App 的名称和预设 URL 填入输入框。

### 5.2 性能与体验验收
*   [ ] **响应速度**：界面交互流畅，无明显卡顿。
*   [ ] **UI/UX**：
    *   界面美观，符合现代 B 端产品风格（卡片式、阴影、圆角）。
    *   加载状态（Loading）有明确的视觉反馈。
    *   移动端适配良好（响应式布局）。
*   [ ] **鲁棒性**：API 请求失败或解析错误时，有友好的错误提示（Toast 或 错误卡片）。
